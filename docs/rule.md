# Fw Rule

## 简介

这份文档记录 `fw/` 框架仓库自身的长期维护规则。

`fw/` 按独立仓库 / submodule 使用，因此框架文档随框架一起维护：

- `fw/README.md`
- `fw/docs/rule.md`
- `fw/docs/spec.md`
- `fw/docs/use.md`

## 规则

- `fw/` 只承载可复用、与具体游戏内容无关的框架能力。
- `fw/` 不放具体游戏玩法、具体 UI 业务和项目私有协议内容。
- 框架维护文档统一放在 `fw/docs/`，宿主仓库不再维护 `docs/fw/`。
- 新游戏接入 `fw/` 时，优先按 `fw/README.md` 走接入流程。
- 当前游戏的接入事实、玩法规则、私有协议和业务说明仍放在宿主仓库对应领域文档里。
